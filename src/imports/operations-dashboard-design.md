Design a multi-page operations analytics dashboard for a quick commerce logistics platform that monitors dark store performance, delivery times, and stockout rates.

The dashboard should look like a premium SaaS analytics product used by logistics operations teams, combining Swiss editorial minimalism with subtle neo-brutalism influences.

The design must feel human-crafted, structured, and product-grade, similar in quality to dashboards from Stripe, Datadog, Linear, and modern logistics control systems.

STRICTLY avoid typical AI-generated UI patterns.

Visual Design Rules
Style

Swiss editorial minimalism + subtle neo-brutalist accents

clean product dashboard aesthetic

strong typography hierarchy

tactile, human-designed layout

Avoid:

gradients

glassmorphism

glowing UI

neon color palettes

generic AI dashboard styles

Color System

Base Background: #0A0A0A (matte black)
Surface Cards: #171717
Primary Text: #F2F2F2
Accent Color: #FF4F00 (industrial orange)

Rules:

accent color only for interactions and highlights

charts should use neutral greys with orange highlights

maintain strong contrast and readability

Apply a 2% grain/noise texture overlay to backgrounds for a subtle physical texture.

Typography

Headers / Section Titles
Inter Tight Bold
Tracking: -2%

Data Labels / Metrics / Tables
Space Mono Regular

Numbers should appear monospaced for readability in analytics tables and KPIs.

Layout Structure

Desktop Frame: 1440 × 900

Grid System:

12 column grid

asymmetric bento-box card layout

width: 240px

fixed position

icons + labels

Sidebar sections:

Dashboard

Dashboard Content

Design the main dashboard as a logistics operations control panel.

Greeting Header

Example:
“Operations Overview”

Include current date and active store filter.

KPI Metric Cards

3 wide KPI blocks:

Average Delivery Time

% Deliveries exceeding 10-minute SLA

Stockout Rate

Each card should show:

large numeric metric

small trend indicator

comparison vs previous period

Charts

Include realistic analytics widgets:

1️⃣ Delivery Time Trend
Line chart showing delivery performance across the day

2️⃣ Orders by Hour
Heatmap or histogram showing order activity

3️⃣ Stockouts by Product Category
Bar chart showing which products frequently go out of stock

4️⃣ Dark Store Comparison
Horizontal bar chart comparing store delivery performance

Charts should be designed as visual placeholders that can later be replaced with Power BI or QuickSight embeds.

Data Table

Large operational table with columns:

Order ID

Store ID

Zone

Order Time

Delivery Time

Delivery Duration

Stockout Flag

Use Space Mono for numeric values.

Interactive UI States

Generate frames showing interactive variants.

Logo Interaction

Top-left logo component.

Hover state:

geometric icon splits into two interlocking shapes

typography weight shifts from Regular → Black

subtle rotation animation

Clicking logo returns to the main dashboard.

KPI Card Hover

Hover effect:

card background abruptly switches to #F2F2F2

text switches to #0A0A0A

sharp 8px solid #FF4F00 offset shadow appears bottom-right

animation duration: ~120ms

Chart Hover Interaction

When hovering a bar:

active bar becomes #FF4F00

other bars drop to 20% opacity

squared tooltip follows cursor

Tooltip includes:

exact value

timestamp or category label

Scroll Reveal Effect

Create a mid-scroll frame:

top dashboard cards fully visible

lower cards shifted down 30px

opacity 40%

simulating staggered scroll reveal animation

Page Transition

Design a page transition wipe animation:

Frame showing a solid #FF4F00 panel covering 60% of the screen from left to right, masking the dashboard content underneath.

This simulates a smooth navigation wipe between dashboard pages.

collapsible sidebar animation

date range picker

loading skeletons for charts

notification slide-out panel

dark mode toggle
